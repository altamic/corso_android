Quesiti corso Android

Cognome

Android OS e’ basato su Kernel:

  1. QNX
  2. Linux
  3. BSD
  4. Windows

Le applicazioni in Android sono eseguite da:

  1. Java Virtual Machine
  2. Emulatore ARM
  3. Dalvik Virtual Machine

In Android OS un’applicazione e’

  1. un’utente distinto
  2. un processo distinto
  3. un thread distinto

Un componente applicativo in Android e’

  1. un Intent
  2. una View
  3. un Layout
  4. un Service

Un RelativeLayout e’

  1. un contenitore di View
  2. uno stile di presentazione
  3. un elemento grafico
  4. un esempio di stile

In un’Activity il metodo onPause() viene chiamato

  1. quando un’altra activity compare in primo piano
  2. quando l’activity ha finito di disegnare l’interfaccia grafica
  3. quando sono concluse le attivita’ in background
  4. quando si vuole arrestare l’esecuzione dell’activity

Un Service in Android potrebbe

  1. eseguire un processo che recupera dati remoti
  2. mostrare l’esito di un’elaborazione remota
  3. eseguire un processo che aggiorna l’interfaccia grafica
  4. aprire una connessione con un database

Un Content Provider in Android serve a

  1. garantire che ci sia una connessione con un database
  2. fornire un meccanismo per condividere dati nelle applicazioni
  3. rendere l’interfaccia ai dati meno uniforme per le applicazioni
  4. nascondere i dettagli implementativi delle connessioni ai dati

Un Broadcast Receiver in Android e’

  1. un raccoglitore di Intent
  2. una sottoclasse di Intent
  3. un servizio che ascolta le notifiche
  4. un componente di ogni applicazione

Un Intent in Android puo’

  1. esporre delle API per modificare i dati
  2. fornire un meccanismo di osservazione sui dati
  3. contenere dati ausiliari
  4. comportasi come un listener

Il Manifest di un’applicazione Android

  1. dichiara i componenti applicativi che saranno usati
  2. indica esclusivamente i permessi dell’utente
  3. puo’ essere compilato programmaticamente
  4. puo’ cambiare nel corso dell’applicazione

In Android le directory di progetto

  1. sono arbitrarie
  2. hanno delle convenzioni sui nomi
  3. possono essere solo quelle fornite dal sistema
  4. riflettono la struttura del pattern MVC

Un attributo di un XML nel namespace Android puo’

  1. indicare un riferimento ad una stringa
  2. fare riferimento ad una procedura Java
  3. trasformarsi in un oggetto Java
  4. includere sinonimi

Un layout parameter in Android serve a

  1. definire dimensioni e posizioni delle View figlie
  2. specificare il comportamento dell’interfaccia grafica
  3. comporre gli elementi delle View
  4. definire gli elementi della View

Un Adapter in Android

  1. fornisce un’interfaccia comune al modello dei dati sottostante ad un widget
  2. converte una view al fine di mostrare dati
  3. fornisce un’interfaccia per i layout
  4. converte un layout al fine di mostrare dati
